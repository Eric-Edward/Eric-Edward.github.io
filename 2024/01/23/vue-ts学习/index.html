选项式和组合式的区别

选项式：vue2的样式，将数据data、方法methods之类的单独进行书写。

组合式：vue3的样式，数据、方法等内容都写在setup中。

==注意，在setup中无法使用this，而在vue2中是可以使用setup的。==

并且vue2和vue3可以混写，并且能够在data、methods通过this访问到setup中的内容。但是setup是访问不到data、methods中的内容的。

setup的语法糖

比较正式的写法：

<script lang="ts">
  export default{
    name:"Person",
    setup(){
      ......
    }
  }
</script>

在这个地方中，我们需要使用name来确定当前组件的名称，并且在setup定义相关的内容。

在使用语法糖之后就会变得很简单，例如下面展示：

<script lang="ts">
  export default{
    name:"Person"
  }
</script>
<script lang="ts" setup>
  ......
</script>

但是我们可以看到这里使用了两个script来完成这个操作，也不是特别的方便。所以，我们可以加入一个插件来使得script标签中可以直接加入一个name的属性来设置我们组件的名称。

我们需要添加vite-plugin-vue-setup-extend，然后在vite.config.ts中引用，并且在plugins中进行添加。

添加了这个插件之后就可以在我们的文件中可以直接使用name属性。

响应式数据

Ref方法

我们如果需要某个变量是响应式的，就需要使用ref将这个变量包裹起来，并且在template中使用的时候直接使用即可，但是在setup中使用时，需要使用name.value的格式，例如下面：

let name = ref("张三")

function handleName(){
  name.value = "李四"
}

此时如果某个操作调用了handleName的方法的话就会使得name变成李四，并且立即在页面上进行响应。

watch监视数据内容

watch一共可以监视4中类型的数据

  1. ref定义的数据
  2. reactive定义的数据
  3. 函数返回一个值（getter函数）
  4. 一个包含上述内容的数组

但是我们总共能够碰到以下5种情况：

  1. 情况一：监视ref定义的基本类型数据，这个很简单。下面给一个具体的样例：

    <script setup name="Person">
import {ref, watch} from "vue";

let sum = ref(0)

function addSum() {
  sum.value++;
}

let stop = watch(sum, (value, oldValue) => {
  console.log(value, oldValue)
  if (value == 5) stop()
})
</script>

  2. 情况二：监视ref定义的对象类型数据，这个有一个点需要注意。==当我们需要对象内部某个属性值的时候，这时的新值和旧值是相同的，因为此时这里的值是通过对象的地址来进行获取的，只修改对象的属性是不会导致对象的地址发生改变的，所以改属性时，新旧值相同。同理的，当我们整体的去修改一个对象时，此时对象的地址就会发生变化，从而可以看到新旧值是不相同的。==下面给出一个具体样例：

    <template>
  <div>
    <h1>姓名:{{ person.name }}</h1>
    <h1>年龄:{{ person.age }}</h1>
    <button @click="changeName">点我修改名称</button>
    <button @click="changeAge">点我修改年龄</button>
    <button @click="changePerson">点我改头换面</button>
  </div>
</template>
<script setup name="Person">
import {ref, watch} from "vue";

let person = ref({
  name: "张三",
  age: 18
})

function changeName() {
  person.value.name = "李四"
}

function changeAge() {
  person.value.age++
}

function changePerson() {
  person.value = {name: "王五", age: 48}
}

watch(person, (newValue, oldValue) => {
  console.log(newValue, oldValue)
}, {deep: true})

</script>

  3. 情况三：监听reactive创建的对象类型的数据，这个和情况二的区别就是：==reactive创建的对象的监听默认是开启了深层次监听的，而且我们是无法通过option进行关闭的。==

  4. 监视ref或reactive定义的【对象类型】中的某个属性，注意以下内容：

    • 若该属性值是一个基本类型，需要写成函数类型，其实就是地址值。
    • 若该属性值是一个对象类型，那么我们可以直接写，也可以写成函数，但是建议写成函数。

  5. 监视多个数据

    ​ 这里就是将多个内容放在一个数组里面，然后旧值和新值都是以数组的形式呈现。

watchEffect

​ 这个内容和watch的作用很想，但是我们在使用这个时，并不需要指明要监视那些对象，他可以自己去分析需要监视那些属性。

标签的Ref属性

​ 我们可以为每一个html元素或者一个组件添加ref标签，来确定的查看或者操作某个元素。

<template>
  <div>
    <h1 ref="h">姓名</h1>
    <button @click="disPlay">点我显示h1</button>
  </div>
</template>
<script setup name="Person">
import {ref} from "vue";

let h = ref();

function disPlay(){
  console.log(h.value)
}

</script>

​ 不过需要注意的是，==在标签上使用的ref需要和js或者ts中的变量命名是需要相同的。==

当我们为一个组件使用ref时，在父组件中是无法看到子组件中js的内容的，除非子组件通过defineExpose来进行暴露。

TS中的接口和范型

​ 我们在自己定于数据时，很容易用着用着就会写错。所以我们可以使用ts的接口和范型来帮助我们。

ts中的接口和数据库定于一张表很像。定义完成之后将这个接口对外暴露即可。

export interface Person {
    id:string,
    name:string,
    age:number
}

​ 如果我们需要让一个数组或者别的容器中的所有的内容都是这个Person类型的话就需要使用范型。其中范型的使用也有两种方式。

  1. 在我们的代码中直接使用范型例如
let persons:Array<Person> = [...]

这样就可以让persons中的每一个人都是符合我们自定义的Person的

  1. 在我们定义的types/index.ts中进行定义

vue3使用props进行参数传递

​ 这里可以有三种方式进行传递，如下所示：

  1. 只接收一个数据。具体情况如下所示

  2. 接收一个数据并且限制这个数据的类型。

    父组件
<template>
  <People :ren="ren1"/>
</template>
<script setup name="Person" lang="ts">
import {type Person} from "@/types";
import People from "@/components/People.vue";
let ren1: Person = {
  name: "Eric1",
  age: 20,
  id: "12345667789"
}
</script>

子组件
<template>
  <h1>姓名:{{ ren.name }}</h1>
  <h1>年龄:{{ ren.age }}</h1>
</template>
<script lang="ts" setup>
import type {Person} from "@/types";
defineProps<{ ren: Person }>()
</script>
<style scoped>

</style>

    我们可以看到，在父组件中传递时需要将ren这个添加上’:’，因为在vue中，加上这个就相当于去进行解析，而不是单纯的字符串。我们在子组件中进行接受的时候，如果要接受的属性必须是某个特定的类型的时候，我们需要给这个属性加上类型的约束（==注意添加约束的时候需要写在范型里面，里面写了之后，外面就不需要再写了==）

  3. 接收一个数据并且为这个数据加上一个默认值。

    <template>
  <h1>姓名:{{ ren.name }}</h1>
  <h1>年龄:{{ ren.age }}</h1>
</template>
<script lang="ts" setup>
import type {Person} from "@/types";
   
withDefaults(defineProps<{ ren?: Person }>(), {
  ren: () => ({name: "eric", age: 20, id: "12345566778888676"})
})
</script>
<style scoped>
   
</style>

    ​ 这里有不少的坑，这个地方的书写相较于vue2麻烦了不止一点半点。我们这里需要使用withDefaults这个关键词来帮助我们设置默认值，而且默认值需要使用肩头函数返回一个对象，我试了好几次才找到返回一个对象的方法，要不就使用一个小括号包起来，要不就在大括号中写一个return然后再加上需要返回的对象即可。

生命周期

vue2

​ vue2主要是这个4个阶段，每个阶段都有两个过程，一个是处理之前，一个是处理完毕。

vue3

​ vue3生命周期的创建阶段时通过setup来完成了，而且我们在使用这个钩子函数的时候需要自己手动的引入这些函数才行。

==注意，不管是vue2还是vue3都不止这几个钩子，还有其他的钩子，我们会在后面的学习中不断的遇到。==

自定义hooks

​ 我们可以自己定义对于不同数据的处理方法，并且把不同的属性和方法封装在一个文件中，形成一个useXXX的hook，我们只需要在需要使用的文件中进行调用即可。同时，这种方法也体现出了==高内聚、低耦合==的思想

vue3路由

路由使用的步骤

​ 在vue3中，路由使用是非常简单的。一共有一下几个步骤

  1. 我们划分好导航区、内容区
  2. 引入路由的内容，如RouterLink、RouterView等
  3. 编写路由的规则，确定好每个路由对应的组件
  4. 合起来

两个注意点

  1. 注意区分路由组件和一般组件，路由组件一般放在pages和views里面，而一般组件通常存放在components里面。==一般组件是直接书写名字来使用，而路由组件使用实时渲染来使用==
  2. 路由组件通常在不使用的时候会被卸载掉，在使用时进行挂载。

路由器的工作模式

​ 路由器的工作模式主要有两种，一种是history模式，一种是hash模式。

==SEO 是搜索引擎优化（Search Engine Optimization）的缩写，是一种通过优化网站内容、结构和其他相关因素，以提高网站在搜索引擎中排名和可见性的技术和策略。SEO 的目标是使网站在搜索引擎中获得更高的排名，从而增加网站的流量、提高用户的访问量和质量，进而达到增加销售、推广品牌或实现其他业务目标的目的。==

to的两种写法

  • 第一种就是to后面添加一个字符串

    <RouterLink to="/home">主目录</RouterLink>

  • 第二种写法如下

    <RouterLink :to={path:'/home'}>主目录</RouterLink>
