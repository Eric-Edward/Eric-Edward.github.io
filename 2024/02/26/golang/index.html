Golang学习笔记心得

学习过程中的一些小知识点

  • Goroutine之间的通信并不是共享一段内存，在这段内存中来保存数据设置锁。而是通过在进程之间通信来实现共享

    Don’t communicate by sharing memory, share memory by communicating

  • 在Google中，搜索一个关键词时，会搜索所有相关的内容（图片、视频等）。而且对于每个类型都会进行多次的搜索，而对于相同类型的内容，我们只采取第一次的数据。从这里我们就可以看出来，我们在通道c收到了一次数据之后就直接返回了。

  • 在每一个源文件中，全局变量会先被定义，然后是执行init（）函数最后是执行main（），所以我们可以在init函数中完成初始化的工作。==注意，如果是别的文件使用我们当前文件中的变量时，在导入包的过程中就会执行当前包中的init()函数，来初始化需要引入的变量。然后如果导入的文件中也有变量的定义和初始化的话，时在需要导入的完成之后在进行处理。==

  • ==匿名函数==

    package main

import "fmt"

func main() {
	ans := func(x, y int) int {
		return x + y
	}(1, 2)

	sub := func(x, y int) int {
		return x - y
	}
	ans2 := sub(20, 1)
	
	fmt.Println(ans, ans2)
}

    从这里我们可以看到，我们可以直接定义一个没有具体名称的函数，并且在函数定义的后面加上小括号进行调用。或者我们可以讲一个匿名函数绑定一个名称，通过这个名称去进行调用。而全局匿名函数的使用就是在文件中直接定义函数，并将函数给到一个变量。==不过用的最多的方法就是直接在函数中进行匿名函数的定义和使用==

  • 闭包的相关内容：在我的理解中，闭包就是一个返回函数类型的函数，但是返回的这个函数可以引用返回她的函数中的变量。==我的理解：我感觉就像是一个服务器在接受到请求之后，现在闭包中做一下响应前的一些操作，然后再去进行响应。==

  • 工厂模式就是，我在一个包中不直接暴露我的结构体，而是通过一个函数来返回一个结构体的指针。这样外部的文件就无法直接的操作我包中定义的结构体了

  • go中的封装、继承和多态

    在go中，==因为go又一个大写字母会从当前文件进行暴露的特性，所以我们可以根据这个特性来进行设计（其实就是通过包来实现封装）==。我们可以创建一个结构体，让我们的属性变量设置小写的字母，这样我们在暴露出结构体的时候，外部用的用户无法直接的操作我们的隐藏的变量。然后我们可以通过方法来获取一个结构体内部的相关属性。

  • Go 的继承是通过在一个结构体中使用另外一个结构体来进行实现的。

    不过需要注意的是，如果这两个结构体的定义是在一个包中的话，那么引用别人的结构体是能够直接访问到被引用的结构体的，但是如果两者不在一个包中，那么只能通过GET函数来进行获取

  • Defer 、panic和recover

    注意defer 有一下三个特点

    1. defer语句中的参数在运行到defer的语句所在行时进行计算

      func a() {
    i := 0
    defer fmt.Println(i)
    i++
    return
}

      例如上面这个案例，最终将会输出0

    2. 延迟函数是一个后进先出的执行过程，例子如下

      func b() {
    for i := 0; i < 4; i++ {
        defer fmt.Print(i)
    }
}

      我感觉因为go的defer是使用stack来保存相关的函数的，所以最后压入栈的会先被执行

    3. 延迟函数可以修改返回值

      func c() (i int) {
    defer func() { i++ }()
    return 1
}

      我们从这个例子中，可以看到c()函数会先返回一个1，然后延迟函数一执行就会把i修改为2。所以我感觉defer 语句如果是一个函数的话，那这个函数就是一个闭包，他会持续的引用原函数中的变量，直至闭包结束。

    Panic：恐慌可以由程序员执行调用，也可以在程序发生运行时错误时出发。恐慌发生时，正常的空自流会中止，但是defer函数会正常的继续执行。然后等到所有的defer语句执行完毕后，如果恐慌没有被恢复，那么会继续向函数的调用者传递恐慌。

    Recover：会中止函数的恐慌，然后正常的执行控制流的内容。
